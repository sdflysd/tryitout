## Browser Tool Preference

- Do not use Codex built-in browser automation tools unless the user explicitly asks for them.
- This includes the in-app Browser plugin, the Chrome extension/browser-client plugin, and `agent.browsers` access through `node_repl`.
- For browser verification, prefer project tests, command-line checks, system Chrome launched by external scripts, Playwright from the project/toolchain, or OS-level screenshots.
- If browser automation is useful but the available option is a Codex built-in browser tool, ask the user for explicit confirmation before using it.

## Local Log Files

- Write ad hoc command logs generated by agents to `output/logs/`.
- Do not create `*.log` files in the project root.
- When a long-running command needs persistent output, redirect stdout/stderr to descriptive files under `output/logs/`, for example `output/logs/dev-server.out.log` and `output/logs/dev-server.err.log`.
